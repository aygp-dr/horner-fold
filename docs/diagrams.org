*Key fact*: GitHub renders mermaid inside =```mermaid= fenced code blocks in
=.md= files. GitHub does *not* render mermaid in =.org= files — the
=#+BEGIN_SRC mermaid= block appears as literal source text. There is no
workaround for this other than tangling to =.mmd= files and rendering PNGs
via mmdc, then including the PNGs with =[[file:...]]= links.

For ob-mermaid: all blocks below use =:file= to invoke mmdc. They also use
=:tangle= to write standalone =.mmd= files for direct mmdc debugging.

* Diagram: Fold/Unfold Duality

Catamorphism (fold) collapses a list into a scalar. Anamorphism (unfold)
generates a list from a seed. The encode/decode pair composes them into a
hylomorphism (round-trip identity).

#+BEGIN_SRC mermaid :file diagrams/fold-unfold-duality.png :tangle diagrams/fold-unfold-duality.mmd :mkdirp t
flowchart LR
    subgraph Catamorphism["Catamorphism (fold)"]
        direction LR
        L1["List\n[a0, a1, ..., an]"]
        S1["Scalar\nfoldl (acc*b + c) 0"]
    end
    subgraph Anamorphism["Anamorphism (unfold)"]
        direction RL
        S2["Scalar\nn"]
        L2["List\nunfoldr (div, mod)"]
    end

    L1 -->|"encode: acc * base + element"| S1
    S2 -->|"decode: quotient / remainder"| L2

    S1 -. "hylomorphism\n(roundtrip identity)" .-> S2

    style Catamorphism fill:#e8f4e8,stroke:#2d7d2d
    style Anamorphism fill:#e8e8f4,stroke:#2d2d7d
#+END_SRC

* Diagram: Horner State Machine (encode/decode "hi" at base 128)

Encoding "hi": h=104, i=105, base=128.

Encode steps: 0 -> 104 -> 13417
Decode steps: 13417 -> 104 remainder 105 -> 0 remainder 104

#+BEGIN_SRC mermaid :file diagrams/horner-state-machine.png :tangle diagrams/horner-state-machine.mmd :mkdirp t
stateDiagram-v2
    state "ENCODE (fold left)" as enc {
        [*] --> E0: start
        E0: acc = 0
        E0 --> E1: read 'h' (104)
        E1: acc = 0 * 128 + 104 = 104
        E1 --> E2: read 'i' (105)
        E2: acc = 104 * 128 + 105 = 13417
        E2 --> [*]: done
    }

    state "DECODE (unfold)" as dec {
        [*] --> D0: start
        D0: n = 13417
        D0 --> D1: divmod(13417, 128)
        D1: quotient = 104, remainder = 105 ('i')
        D1 --> D2: divmod(104, 128)
        D2: quotient = 0, remainder = 104 ('h')
        D2 --> [*]: n = 0, done
    }

    enc --> dec: roundtrip
    note right of enc: catamorphism\nlist -> scalar
    note right of dec: anamorphism\nscalar -> list
#+END_SRC

* Diagram: Cross-Dialect Comparison

Quadrant chart placing dialects on two axes:
- X axis: fold argument order (element-first vs accumulator-first)
- Y axis: threading macro support (none vs built-in)

#+BEGIN_SRC mermaid :file diagrams/dialect-comparison.png :tangle diagrams/dialect-comparison.mmd :mkdirp t
quadrantChart
    title Lisp Dialect Fold Characteristics
    x-axis "(element, acc) order" --> "(acc, element) order"
    y-axis "No threading macros" --> "Built-in threading macros"
    quadrant-1 "acc-first + threading"
    quadrant-2 "elem-first + threading"
    quadrant-3 "elem-first + no threading"
    quadrant-4 "acc-first + no threading"
    Clojure: [0.85, 0.95]
    Janet: [0.85, 0.80]
    Racket: [0.20, 0.65]
    Elisp-dash: [0.80, 0.50]
    Chez: [0.85, 0.15]
    Guile-rnrs: [0.80, 0.10]
    Guile-srfi1: [0.15, 0.10]
#+END_SRC

Notes on placement:
- Guile (SRFI-1) is element-first, no threading -> bottom left
- Guile (rnrs) is acc-first, no threading -> bottom right
- Chez is acc-first, no threading -> bottom right
- Racket is element-first but has ~> (via threading lib) -> upper left area
- Elisp is acc-first, has ->> via dash (external) -> middle right
- Clojure is acc-first, ->> -> as-> all built-in -> top right
- Janet is acc-first, -> ->> built-in -> upper right
